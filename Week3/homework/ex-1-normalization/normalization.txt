1-What columns violate 1NF?

- food_code and food_description columns violate this rule.

The rule: Prevent duplicate records (by applying primary keys):
- There is no primary keys, the values are repeated.

The rule: Attribute domain should not change (all values in a column must be of the same kind or type).
- dinner_date column data are not the same type.
